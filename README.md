# 決戰台北橋 — 機車瀑布鑽車縫模擬器

台北橋每天早晨數千輛機車同時衝過停止線的景象，在台灣被稱為「機車瀑布」。
本專案用 Boids 演算法模擬這個現象，並加入台灣特有的騎士人格與行人類型——道路是大家的，模擬中也有弱勢行人在車陣中掙扎求生。

## Demo

直接用瀏覽器開啟 `index.html`（或架在任何靜態伺服器上）即可執行，無需安裝套件。

## 控制項

| 控制 | 說明 |
|------|------|
| 生成車流量 | 每秒生成的機車數量（1–30） |
| 急躁/鑽車型 比例 | aggressive 機車佔比 |
| 三寶/隨機偏移 比例 | clueless 機車佔比 |
| 行人流量 | 每分鐘從斑馬線生成的行人數（1–10） |
| 討厭行人比例 | 無視號誌、滑手機的行人佔比 |
| 交通號誌覆寫 | 強制切換紅燈/綠燈，或回到自動模式 |

## 三種機車人格

| 類型 | 行為 |
|------|------|
| 普通（白色安全帽） | 正常跟車、遵守號誌 |
| 鑽車型（紅色安全帽） | 超速、加速闖黃燈、縮短安全距離 |
| 三寶（紫色安全帽） | 慢速、左右亂飄、發呆忘記起步、車頭掛菜籃 |

## 四種行人類型

| 類型 | 行為 |
|------|------|
| 一般（黃色頭） | 紅燈穿越，看到車快跑 |
| 長輩（紫色衣） | 移動極慢，跑也跑不快，駝背拐杖 |
| 輪椅（橘色衣） | 最慢，機車迴避距離加倍 |
| 討厭（綠色衣） | 無視號誌、滑手機、不逃 |

## 技術架構

```
index.html          主頁面與控制面板（Tailwind CSS）
assets/js/
  main.js           主迴圈、canvas resize、UI 事件
  Motorcycle.js     機車實體：物理更新、Boids 行為邏輯
  ScooterModel.js   純渲染：依人格畫出不同外觀
  Environment.js    交通號誌狀態機（綠→黃→紅自動循環）
  Pedestrian.js     行人實體：四種類型、動態生成/消滅
  Vector2D.js       2D 向量數學工具
  QuadTree.js       空間分割，O(n log n) 鄰近偵測
  config.js         全域參數設定
```

## 已知限制

- 行人 AI 為線性掃描，行人數量極大時略有效能消耗（目前流量範圍內無影響）
